# TextLens

**A free, open-source alternative to expensive AI-content and plagiarism detection tools.**

> Paid AI-detectors and plagiarism checkers are often locked behind subscriptions — even for checking your own writing before submission. TextLens gives you a transparent, self-hostable way to get the same kind of signal, for free.

## Features

- **AI-Likelihood Detection** — estimates how likely a piece of text was generated by an LLM, using perplexity and burstiness analysis (the same family of signal used by tools like GPTZero), with sentence-level highlighting.
- **Similarity / Plagiarism Check** — compares two texts using TF-IDF cosine similarity and highlights overlapping phrases.

## Tech Stack

| Layer | Tech |
|---|---|
| Frontend | Next.js, TypeScript, Tailwind CSS |
| Backend | FastAPI (Python) |
| ML / NLP | Hugging Face Transformers (GPT-2), scikit-learn (TF-IDF) |
| Deployment | Vercel (frontend), Render (backend) |

## How It Works

### AI Detection
Text is scored with GPT-2 to compute:
- **Perplexity** — how "surprised" the model is by the text. Human writing tends to have higher, more variable perplexity than LLM output, which is often more "predictable."
- **Burstiness** — variance in per-sentence perplexity. Human writing fluctuates in complexity; AI writing tends to be more uniform.

These combine into an AI-likelihood score (0–100), with sentences color-coded by how "AI-like" they appear.

### Similarity Check
Two texts are vectorized with TF-IDF and compared via cosine similarity. Matching 6-word phrases are extracted and shown as direct overlaps.

> Both methods are heuristic estimates, not definitive verdicts — presented transparently as such.

## Getting Started

### Backend
```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```
API docs: `http://localhost:8000/docs`

### Frontend
```bash
cd frontend
npm install
cp .env.local.example .env.local
npm run dev
```
Visit `http://localhost:3000`
